# DTCG and YAML Design Tokens Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Highlight DTCG 2025.10 structured colors and aliases in JSON/JSONC/YAML, including opt-in trusted relative cross-file `$ref` resolution.

**Architecture:** Parse JSON-family and YAML syntax into one ranged token-entry model, resolve local semantics synchronously, and layer an async workspace loader only when cross-file references are enabled. Convert validated structured colors through existing color utilities so every consumer continues receiving canonical `ColorMatch` values.

**Tech Stack:** TypeScript, `jsonc-parser` 3.3.1, `yaml` 2.9.0, VS Code Workspace FS, Vitest, pnpm.

## Global Constraints

- Use pnpm; prefix shell commands with `rtk`, except `pnpm typecheck`.
- Use RED→GREEN TDD for production behavior.
- Preserve synchronous JSON strategy results unless cross-file resolution is enabled.
- Preserve legacy JSON/JSONC string modes and existing source ranges.
- Default `resolveDesignTokensAcrossFiles` to `false` and require workspace trust.
- Support desktop, Web, virtual workspaces, open unsaved dependencies, and 512 KiB external-file limits.
- Commit the complete feature as `feat: support DTCG and YAML design tokens`.
